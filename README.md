FastAPI Product Parser 🛒

Асинхронний REST API сервіс для парсингу товарів та відгуків з українських маркетплейсів (Hotline, Comfy, Brain) згідно з тестовим завданням.

🚀 Технології

FastAPI — Роутинг, валідація даних, автоматична генерація OpenAPI/Swagger документації.

Playwright (Async) — Використовується для рендерингу динамічного контенту (SPA/React/Vue) та обходу базового захисту від ботів (Cloudflare) на сайті Hotline.

HTTPX & BeautifulSoup4 — Швидкий парсинг статичного HTML та прихованих JSON-LD (SEO) даних для сайтів Comfy та Brain.

MongoDB + Motor — Асинхронне збереження розпарсених даних (офери та коментарі).

Pydantic — Строга типізація та валідація вхідних параметрів і вихідних відповідей (Response Models).

Docker & Docker Compose — Повна контейнеризація застосунку (API + MongoDB) для швидкого розгортання.

🏗 Архітектура та Патерни

Strategy Pattern (Парсери): Реалізовано базовий абстрактний клас BaseParser, від якого успадковуються HotlineParser, ComfyParser та BrainParser. Це дозволяє легко додавати нові магазини в майбутньому.

Dependency Injection / Factory: Маршрутизатор (Router) автоматично визначає необхідний парсер (Comfy або Brain) на основі переданого URL.

Lifespan Events: Використано сучасний підхід FastAPI для керування підключенням до бази даних при старті та зупинці сервера.

Обробка винятків (Error Handling): Парсери маплять специфічні помилки (наприклад, блокування Cloudflare або таймаути) у відповідні HTTP-статуси (429 Too Many Requests, 408 Request Timeout, 500 Internal Server Error).

⚙️ Запуск проєкту

Клонувати репозиторій:

git clone <посилання-на-твій-репозиторій>
cd fastapi-product-parser


Запустити проєкт за допомогою Docker Compose:

docker-compose up -d --build


API буде доступне за адресою:
http://localhost:8000

Інтерактивна документація (Swagger UI):
http://localhost:8000/docs

📡 API Ендпоінти

1. Отримання оферів товару

Отримує список пропозицій магазинів для заданого товару на Hotline.

Запит: GET /product/offers

url (обов'язковий): URL сторінки товару на Hotline.

timeout_limit (опціональний): Максимальний час очікування (в секундах).

count_limit (опціональний): Обмеження кількості повернутих оферів.

price_sort (опціональний): Сортування за ціною (asc або desc).

Поведінка: Результат автоматично зберігається у колекцію hotline_products в MongoDB.

2. Отримання відгуків про товар

Отримує список коментарів та відгуків про товар із сайтів Comfy або Brain. Джерело визначається автоматично по URL.

Запит: GET /product/comments

url (обов'язковий): URL сторінки товару на Comfy або Brain.

date_to (опціональний): Дата у форматі YYYY-MM-DD, до якої потрібно парсити відгуки.

Поведінка: Результат автоматично зберігається у колекцію product_comments в MongoDB.

🛡 Обробка специфіки сайтів

Hotline: Оскільки сайт сильно захищений та динамічний, використовується Headless Chromium через Playwright для імітації поведінки реального користувача (скролінг, очікування завантаження мережі).

Comfy / Brain: Використовується легковаговий httpx. Для Comfy реалізовано витягування прихованих відгуків із SEO-блоків application/ld+json, оскільки основний контент рендериться через JS.
